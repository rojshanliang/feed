SPEC · 广场页结构说明

顶部导航 NavBar（广场 / 更多），标题左对齐 · 广场为底Bar根页面，无上一页可返回，不展示返回箭头
底 Bar 视频 / 图文 / 内容发布 / 广场（当前） / 我的
页面风格 类朋友圈动态流 · 单列时间倒序，聚合展示多用户动态（非单人主页）
信息展示 文字图片视频 三种内容形式，可单独或与文字搭配出现
创作者等级标签（新增） 昵称右侧紧跟一个等级徽章（如 Lv.5），黄色描边+暖色底，弱化于昵称本身；卡片内展开的点赞/打赏头像墙同样展示各用户的等级
话题标签 纯文字 #话题 内联紧跟正文展示，不单独圈出/不带背景色胶囊，与图文首页/详情页话题样式一致（见"阿强"摆摊动态）
图片动态 1 张时单图大图展示（4:3），多张时 3 列九宫格（1:1 裁切）
视频动态 竖版视频缩略图（9:16）+ 右下角时长标签，点击进入播放
功能按钮位置 「···」与文字/图片/视频内容并列同一行、居右对齐，垂直对齐到内容最后一行（图片/视频则贴内容底部），位于点赞·评论·打赏展示区域之上
互动行为 点击「···」向下横向展开一排图标菜单（参考微信朋友圈样式），内含 点赞评论打赏 三项操作，避免向上展开导致列表首条被遮挡
点赞可取消 菜单内「点赞」为可点击开关：未点赞 ♡ 点击变已点赞 ❤（数字 +1）；已点赞再点一次则取消（数字 -1），与详情页点赞逻辑一致
已互动用户展示 有点赞/打赏时，「···」按钮行下方常驻浅灰底信息条展示具体用户、打赏的礼物名称与数量（非现金金额）；无互动则不展示该区块；点击点赞/打赏这两行，卡片内直接展开完整名单（不跳转页面）
评论展示（有条件） 若该动态有评论，在信息条内追加分隔线 + 评论预览（昵称+内容）+「查看全部 N 条评论」；无评论则不展示；点击「查看全部」同样在卡片内展开完整评论列表
零互动态 无点赞/评论/打赏时，内容下方不展示任何信息条，仅保留「···」入口（见最后一条动态）
② 评论中态 点击「···」菜单「评论」后，目标动态左侧加灰色竖线 + 浅灰底标记（隐藏其「···」）· 底Bar 被评论输入条替换，输入框自动聚焦并弹出键盘
评论输入条 头像 + 输入框（含闪烁光标，占位文案随场景变化，如「回复 阿伟」）+ 表情按钮 +「发送」按钮，与图文详情页评论输入条同规格
⑧ 表情面板中 点击发送按钮旁的表情图标后，原本弹出的模拟键盘被替换为表情面板（同一位置切换），再点一次表情按钮切回键盘；选中的表情插入到输入框光标处
表情面板 8 列网格铺开常用表情，点击任意表情追加到输入内容末尾，不关闭面板，方便连续插入多个表情
③ 打赏中态 点击「···」菜单「打赏」后，动态流整体变暗，底部弹出虚拟礼物选择弹层（与同城定位失败弹层结构一致）· 打赏的是虚拟礼物，计价直接用真实货币数值，不引入虚拟币/余额换算
打赏弹层 标题「打赏给 [创作者昵称]」+ 8 款虚拟礼物图标胶囊（4 列 × 2 行，图标+名称+价格数字，不带货币符号，默认选中一档，超出两行可左右滑动/翻页）+「打赏 [礼物名] N」按钮
④ 图片浏览态 点击动态里任意一张图片后进入全屏图片浏览层（黑底），可左右滑动切换，与图文详情页查看大图结构一致
图片浏览层 顶部左上角关闭按钮 + 图片舞台 + 底部页码「N / 总数」与指示点，点击关闭返回动态流原位置
⑤ 视频播放态 点击动态里的视频缩略图后进入全屏竖版播放，覆盖整个小程序屏幕，与视频 Feed 播放页风格一致
视频播放层 顶部左上角关闭按钮 + 播放画面 + 居中暂停/播放按钮 + 左下角创作者头像昵称与文案 + 底部进度条
⑥ 打赏成功态 点击「打赏 [礼物名] N」按钮且请求成功后，打赏弹层收起，居中浮现成功 Toast（绿色✓），已互动信息条同步追加「你刚刚打赏」记录
成功 Toast 绿色圆形✓图标 + 「打赏成功」标题 + 礼物名称与数量的确认文案，无需用户操作，短暂停留后自动淡出
⑦ 打赏失败态 请求失败（如网络异常）时，打赏弹层保留原选中的礼物不清空，叠加显示失败 Toast，方便用户直接重试
失败 Toast 红色圆形✕图标 + 「打赏失败」标题 + 失败原因文案 + 「重试」按钮，点击重试重新发起同一笔打赏请求
卡片内互动信息展开（新增） 点击信息条内的「点赞」行、「打赏」行，或「查看全部 N 条评论」，不跳转页面、不打开弹层，直接在该条动态卡片内向下展开完整互动信息，原本的截断预览（.moment-social）隐藏
展开内容结构 · 不分 Tab 依次为「点赞头像墙 → 打赏头像墙 → 评论列表」，三块同时全部直接展示（仿微信朋友圈，直给式呈现），不做 Tab 切换；有几类数据就展示几个分区，分区间用细分隔线隔开
点赞头像墙（仅头像，无昵称） 展示全部点赞用户头像，头像下方不展示昵称文字，纯头像密排；不做「+N」截断，也不设内部滚动框，随页面自然向下延展；头像采用微信朋友圈式圆角正方形（非圆形）
打赏头像墙（头像 + 昵称） 展示全部打赏用户，头像下方保留昵称，用于分辨具体是谁打赏；头像右下角额外叠加所送礼物角标；不做「+N」截断
评论列表（展开态，完整展示） 展示该条动态的全部评论，不做省略、不设内部滚动；昵称+等级+完整评论正文同行紧凑排布，仅单层展示（不支持二级评论嵌套）
⑨ 互动信息展开态（新增，独立状态） 在顶部状态切换栏新增独立入口，单独展示某条动态的完整展开效果（点赞头像墙→打赏头像墙→评论列表依次铺开），无需先操作默认态再点击触发
收起 「收起」置于展开内容最上方、右对齐，与触发展开的「点赞/查看全部评论」文字入口同一视觉量级（纯文字，非独立按钮），点击后就地收回展开区域，恢复为原本的点赞/打赏/评论摘要预览；无需先滚动到展开内容末尾才能收起
