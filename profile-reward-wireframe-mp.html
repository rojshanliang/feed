SPEC · 打赏额度模块结构说明（个人页 / 创作者页）

打赏额度卡 位于数据四格（关注/粉丝/获赞/内容）下方、操作按钮上方 · 独立卡片，1px 描边 + 6px 圆角
数据展示 七日打赏额度总打赏额度 两项并列，金额用等宽数字（IBM Plex Mono），不带货币符号（打赏为虚拟礼物，非真实货币）
不重复展示 页面本身已展示头像与昵称，打赏卡内不再重复出现创作者信息
适用范围 个人页创作者页 均展示，数值随粉丝量级同比放大
与原页面差异 在 profile-content-tab-wireframe-mp.html 基础上新增本模块，其余结构保持一致

SPEC · 广场 Tab 结构说明（个人页 / 创作者页）

Tab 位置 内容 Tab 由「视频 / 图文」扩展为「视频 / 图文 / 广场」三个 Tab，广场为第三项
动态形式 纯文字图片+文字视频+文字 类朋友圈动态流，单列时间倒序排列
图片动态 3 列图片九宫格布局（本例展示 6 图），单张图片 1:1 裁切
视频动态 竖版视频缩略图（9:16）+ 右下角时长标签，点击进入播放
已互动用户展示 内容下方常驻一块浅灰底信息条，分两行展示「已点赞用户」和「已打赏用户」，打赏行展示对方所送的虚拟礼物徽章（与广场首页打赏逻辑一致，非金额），点赞行超出人数用「等 N 人点赞」收尾
评论展示（有条件） 若该动态有评论，在已互动用户信息条内加一条分隔线，展示 1-2 条评论预览（昵称+内容）+「查看全部 N 条评论」；无评论则不展示该区块，点击「···」→评论仍可发表首条评论
互动入口 已互动用户展示条下方默认仅展示「···」更多按钮，不直接铺开点赞/评论/打赏数字
互动能力 点击「···」展开悬浮菜单，菜单内含 点赞评论打赏 三项操作；点击菜单外区域自动收起
点赞 / 打赏名单弹层（新增） 点击「已点赞用户」/「已打赏用户」信息行，底部弹出完整名单弹层，展示全部互动用户头像+昵称；打赏名单每个头像右下角叠加对应礼物图标，区分谁送了什么礼物，与 square-detail-wireframe-mp.html 的头像墙结构一致
适用范围 个人页创作者页 均展示，创作者页动态互动量级更高
