SPEC · 打赏额度模块结构说明（个人页 / 创作者页）

打赏额度卡 位于数据四格（关注/粉丝/获赞/内容）下方、操作按钮上方 · 独立卡片，1px 描边 + 6px 圆角
数据展示 七日打赏额度总打赏额度 两项并列，金额用等宽数字（IBM Plex Mono），不带货币符号（打赏为虚拟礼物，非真实货币）
不重复展示 页面本身已展示头像与昵称，打赏卡内不再重复出现创作者信息
适用范围 个人页创作者页 均展示，数值随粉丝量级同比放大
与原页面差异 在 profile-content-tab-wireframe-mp.html 基础上新增本模块，其余结构保持一致；个人页移除「编辑个人资料」按钮

SPEC · 广场 Tab 结构说明（个人页 / 创作者页）

Tab 位置 内容 Tab 由「视频 / 图文」扩展为「视频 / 图文 / 广场」三个 Tab，广场为第三项
动态形式 纯文字图片+文字视频+文字 类朋友圈动态流，单列时间倒序排列
图片动态 3 列图片九宫格布局（本例展示 6 图），单张图片 1:1 裁切
视频动态 竖版视频缩略图（9:16）+ 右下角时长标签，点击进入播放
功能按钮（···）位置 与文字/图片/视频内容并列同一行、居右对齐，垂直对齐到内容最后一行（图片/视频则贴内容底部）——与 square-wireframe-mp.html 广场首页结构完全一致
互动能力 点击「···」向下横向展开一排图标菜单（微信朋友圈样式），内含 点赞评论打赏 三项操作；点击菜单外区域自动收起
已互动用户展示 内容下方常驻一块浅灰底信息条，分行展示「已点赞用户」和「已打赏用户」，打赏行展示对方所送的虚拟礼物徽章（非金额）；若有评论，同一信息条内追加分隔线 + 评论预览 +「查看全部 N 条评论」
卡片内互动信息展开（与广场首页一致） 点击「已点赞用户」行、「已打赏用户」行，或「查看全部 N 条评论」，不跳转页面、不打开弹层，直接在该条动态卡片内向下展开完整互动信息：点赞头像墙 → 打赏头像墙 → 评论列表依次堆叠、全部直接展示，不做「+N」截断；头像采用微信朋友圈式圆角正方形
收起 「收起」置于展开内容最上方、右对齐，与触发展开的文字入口同一视觉量级，点击后就地收回，恢复为原本的摘要预览
适用范围 个人页创作者页 均展示，创作者页动态互动量级更高
