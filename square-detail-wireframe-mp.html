SPEC · 广场详情页结构说明（微信朋友圈式单条动态详情）

设计基调 对标微信朋友圈单条动态详情，而非小红书式内容详情：无大头图 Hero、无关注按钮，头像+昵称紧贴正文，互动信息收纳在一个灰底信息框内
页面定位 承载广场单条动态的完整详情，主要入口为话题页卡片点击、广场列表卡片点击
来源提示 若从话题页进入，顶部展示「来自话题 #xxx」提示条，点击可跳回该话题页；广场首页直接进入则不展示
头像 + 昵称 头像采用圆角方形（微信式），与昵称、正文、媒体内容左对齐、一体化排布，不设独立的大号创作者信息行，也不设关注按钮
内容呈现 文字图片视频 三种形式，与广场发布页支持的格式一致，可与文字任意组合；图片/视频均为紧凑缩略图，作为正文的自然延续，而非满屏大图
图片呈现 紧凑 3 列缩略图（64px/格，1 张时单图放大到 130px），点击任意图片进入全屏浏览层，可左右滑动切换
视频呈现 紧凑竖版缩略图（108px 宽）+ 时长角标，点击进入全屏播放（播放层复用视频 Feed 播放页，本图未展开）
「···」功能入口 时间戳同一行右侧的「···」按钮，横向展开 点赞评论打赏 三个入口（与广场首页动态卡片交互一致），取代原先常驻的底部交互条
互动信息展示框（新增） 正文下方统一收纳进一个浅灰圆角信息框，顶部小三角指向「···」；框内依次为「❤ 点赞头像墙」「🎁 打赏头像墙」「评论列表」，视觉上是同一整体
点赞 / 打赏头像墙 展示全部点赞 / 打赏用户头像（圆角方形小头像，26px），超出显示数量后以「+N（共 N 人）」收尾；打赏头像右下角叠加对应礼物图标，区分谁送了什么礼物
评论展示（仅单层） 「昵称：正文」同行紧凑排布，不展示头像，不支持二级评论/嵌套回复；点击「回复」仅把对方昵称带入底部输入框（@昵称），新发表的回复作为独立一条评论追加在列表末尾
评论中态 点击「···」里的「评论」或某条评论的「回复」，底部弹出评论输入条 + 模拟键盘（复用广场首页评论态结构），无需常驻显示
打赏交互 点击「···」里的「打赏」，底部弹出虚拟礼物选择弹层（与广场页打赏弹层结构一致），确认后头像会追加到「打赏头像墙」
