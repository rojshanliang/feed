SPEC · 话题页「广场」Tab 结构说明

Tab 结构调整（本次核心变更） 内容 Tab 由「推荐 / 最新」二项，扩展为「推荐 / 最新 / 广场」三项；广场为新增第三项，不与推荐/最新融合展示
推荐 / 最新（不再混排广场内容） 双列瀑布流仅展示 图文卡视频卡 两种类型，广场来源的文字/图文/视频动态已全部移出，不再与图文/视频模块内容混合展示
来源角标（恢复为原有双来源约定） 由于广场内容已移出，无需再区分三种来源：图文 标注来自图文模块（原有约定），视频模块内容不加角标（原有约定），与融合广场之前的规则保持一致
广场 Tab（新增） 承载该话题下所有带 #话题 发布的广场动态（文字 / 图文 / 视频三种形式），单列朋友圈式 Feed，与瀑布流双列布局完全独立
广场 Tab 展示风格（与广场首页一致） 卡片结构、头像+昵称+等级标签、话题内联展示、图片九宫格/视频缩略图、已互动信息条、「···」横向弹出菜单，均与 square-wireframe-mp.html 广场首页完全一致，直接复用同一套组件与交互逻辑
卡片内互动信息展开（与广场首页一致） 点击「已点赞用户」行、「已打赏用户」行，或「查看全部 N 条评论」，直接在该条动态卡片内展开完整互动信息：点赞头像墙 → 打赏头像墙 → 评论列表依次堆叠、全部直接展示，不做「+N」截断；「收起」置于展开内容最上方
顶部结构不变 话题封面、话题名称、内容/参与数据统计行在三个 Tab 下保持不变，仅 Tab 下方的内容区域随 Tab 切换
点击行为 图文卡进入图文详情页 · 视频卡进入视频播放页 · 广场 Tab 内动态点击进入 square-detail-wireframe-mp.html 广场详情页（均未在本图展示）
